# Project File Specification

This document defines the format and validation rules
for RustWerk's project file (`.rustwerk/project.json`).

## Overview

All project state is stored in a single JSON file at
`.rustwerk/project.json`, relative to the project root.
RustWerk locates this directory by walking up from the
current directory, similar to how git finds `.git/`.

The file uses pretty-printed JSON. Fields that are null
or empty are omitted to keep the file compact and
readable.

## Root Object

```json
{
  "metadata": { ... },
  "tasks": { ... },
  "next_auto_id": 1,
  "developers": { ... }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `metadata` | object | yes | Project metadata |
| `tasks` | object | yes | Map of task ID to task |
| `next_auto_id` | integer | yes | Counter for auto-generated task IDs |
| `developers` | object | no | Map of developer ID to developer (omitted if empty) |

## Metadata

```json
{
  "name": "My Project",
  "description": "Optional description",
  "created_at": "2026-04-02T10:00:00Z",
  "modified_at": "2026-04-02T10:00:00Z"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | yes | Project name (non-empty after trim) |
| `description` | string | no | Project description (omitted if not set) |
| `created_at` | string | yes | Creation timestamp (RFC 3339, UTC) |
| `modified_at` | string | yes | Last modification timestamp (RFC 3339, UTC) |

The `modified_at` timestamp is updated automatically
whenever the project file is saved.

## Tasks

The `tasks` field is a JSON object mapping task IDs
(strings) to task objects. Keys are sorted
alphabetically.

### Task ID Format

Task IDs come in two forms:

- **Mnemonic** (user-supplied): alphanumeric characters,
  hyphens, and underscores. Stored in uppercase.
  Pattern: `^[A-Z0-9_-]+$`.
  Examples: `AUTH-LOGIN`, `DB-SCHEMA`, `T1_SETUP`.

- **Auto-generated**: format `T` followed by a
  four-digit zero-padded number. Generated by
  incrementing `next_auto_id`.
  Examples: `T0001`, `T0042`.

### Task Object

```json
{
  "title": "Implement login",
  "description": "User authentication endpoint",
  "status": "in_progress",
  "dependencies": ["DB-SCHEMA"],
  "effort_estimate": "5H",
  "complexity": 5,
  "assignee": "alice",
  "effort_entries": [
    {
      "effort": "2.5H",
      "developer": "alice",
      "timestamp": "2026-04-03T10:00:00Z",
      "note": "Initial implementation"
    }
  ]
}
```

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `title` | string | yes | — | Task title (non-empty after trim) |
| `description` | string | no | omitted | Task description |
| `status` | string | no | `"todo"` | Task status (see [Status Values](#status-values)) |
| `dependencies` | array | no | omitted | List of task IDs this task depends on |
| `effort_estimate` | string | no | omitted | Effort estimate (see [Effort Format](#effort-format)) |
| `complexity` | integer | no | omitted | Complexity score, 1–1000 |
| `assignee` | string | no | omitted | Developer ID of the assigned developer |
| `effort_entries` | array | no | omitted | Logged effort entries (see [Effort Entries](#effort-entries)) |

### Status Values

The `status` field accepts one of these values
(serialized in snake_case):

| Value | Description |
|-------|-------------|
| `todo` | Not yet started (default) |
| `in_progress` | Currently being worked on |
| `blocked` | Waiting on dependencies or other blockers |
| `done` | Completed |
| `on_hold` | Paused |

**Valid status transitions** (enforced unless using
`--force`):

```
todo        → in_progress, on_hold
in_progress → done, blocked, on_hold
blocked     → in_progress, todo
on_hold     → todo, in_progress
done        → (terminal — no valid transitions)
```

### Dependencies

The `dependencies` array lists task IDs that must be
completed before this task can start.

**Constraints:**
- Each referenced task ID must exist in the project.
- Self-references are rejected.
- Cycles are detected and rejected (depth-first search).
- Duplicate edges are silently ignored (idempotent).

### Effort Format

Effort values are strings with a numeric amount followed
by a unit letter. The numeric part can be fractional
(e.g., `2.5H`). Whole numbers omit the decimal
(e.g., `5H` not `5.0H`).

| Unit | Meaning | Hours |
|------|---------|-------|
| `H` | Hours | 1 |
| `D` | Days | 8 |
| `W` | Weeks | 40 |
| `M` | Months | 160 |

**Constraints:**
- Value must be positive and finite.
- Unit letter is case-insensitive on input, uppercase
  in stored form.

Examples: `5H`, `2.5D`, `0.5W`, `1M`.

### Effort Entries

Each effort entry records actual time spent on a task.
Effort can only be logged on tasks with status
`in_progress`.

```json
{
  "effort": "2.5H",
  "developer": "alice",
  "timestamp": "2026-04-03T10:00:00Z",
  "note": "Debugging auth flow"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `effort` | string | yes | Amount of effort (same format as estimates) |
| `developer` | string | yes | Developer ID who did the work |
| `timestamp` | string | yes | When the effort was logged (RFC 3339, UTC) |
| `note` | string | no | Optional note (omitted if not set) |

## Developers

The `developers` field is a JSON object mapping developer
IDs (strings) to developer objects. Keys are sorted
alphabetically. Omitted entirely if no developers are
registered.

### Developer ID Format

Developer IDs are alphanumeric with hyphens and
underscores. Stored in lowercase.
Pattern: `^[a-z0-9_-]+$`.
Examples: `alice`, `bob-smith`, `dev_lead`.

### Developer Object

```json
{
  "name": "Alice Smith",
  "email": "alice@example.com",
  "role": "project-lead",
  "specialties": ["rust", "backend"]
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | yes | Display name (non-empty after trim) |
| `email` | string | no | Email address (omitted if not set) |
| `role` | string | no | Role on the project (omitted if not set) |
| `specialties` | array | no | List of specialty strings (omitted if empty) |

## Validation Summary

| Item | Rule |
|------|------|
| Project name | Non-empty after trim |
| Task title | Non-empty after trim |
| Task ID (mnemonic) | `^[A-Z0-9_-]+$`, non-empty |
| Task ID (auto) | `T` + four-digit zero-padded number |
| Developer ID | `^[a-z0-9_-]+$`, non-empty |
| Developer name | Non-empty after trim |
| Complexity | Integer, 1–1000 |
| Effort value | Positive, finite number |
| Effort unit | `H`, `D`, `W`, or `M` |
| Status | `todo`, `in_progress`, `blocked`, `done`, `on_hold` |
| Dependencies | No self-references, no cycles, IDs must exist |
| Assignee | Must be a registered developer ID |
| Effort logging | Task must be `in_progress` |

## Examples

### Minimal Project

A freshly initialized project with no tasks:

```json
{
  "metadata": {
    "name": "My Project",
    "created_at": "2026-04-02T10:00:00Z",
    "modified_at": "2026-04-02T10:00:00Z"
  },
  "tasks": {},
  "next_auto_id": 1
}
```

### Full-Featured Project

A project with tasks, dependencies, developers, and
effort tracking:

```json
{
  "metadata": {
    "name": "WebApp",
    "description": "Web application project",
    "created_at": "2026-04-02T10:00:00Z",
    "modified_at": "2026-04-03T15:30:00Z"
  },
  "tasks": {
    "API-BUILD": {
      "title": "Build REST API",
      "status": "in_progress",
      "dependencies": ["DB-SCHEMA"],
      "effort_estimate": "3D",
      "complexity": 8,
      "assignee": "alice",
      "effort_entries": [
        {
          "effort": "4H",
          "developer": "alice",
          "timestamp": "2026-04-03T10:00:00Z",
          "note": "Set up endpoints"
        }
      ]
    },
    "DB-SCHEMA": {
      "title": "Design database schema",
      "status": "done",
      "effort_estimate": "2D",
      "complexity": 5
    },
    "DEPLOY": {
      "title": "Deploy to staging",
      "status": "todo",
      "dependencies": ["TEST-ALL"],
      "effort_estimate": "4H",
      "complexity": 2
    },
    "TEST-ALL": {
      "title": "Write tests",
      "status": "todo",
      "dependencies": ["API-BUILD", "UI-BUILD"],
      "effort_estimate": "1D",
      "complexity": 3
    },
    "UI-BUILD": {
      "title": "Create frontend",
      "status": "todo",
      "dependencies": ["API-BUILD"],
      "effort_estimate": "2D",
      "complexity": 5
    }
  },
  "next_auto_id": 1,
  "developers": {
    "alice": {
      "name": "Alice Smith",
      "email": "alice@example.com",
      "role": "lead"
    },
    "bob": {
      "name": "Bob Jones"
    }
  }
}
```
